# Brand Mark Rollout Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the app's placeholder conic-gradient header mark with the real interlocked-rings brand mark from the "Glowbraid Logo Options" design, and generate real favicon/app-icon assets (currently generic/unstyled), wiring them into the app's `<head>`.

**Architecture:** One SVG mark (three concentric glowing rings, radii/colors below) becomes the single source of truth, expressed three ways: (a) a React component for the live header, (b) standalone static SVG files for contexts with no current UI home, (c) rasterized PNG/ICO tiles for favicon/app-icon slots, generated by screenshotting an HTML harness in a real browser (Playwright) since this machine's ImageMagick has no working SVG delegate (`rsvg-convert` missing — verified: rasterizing SVG directly with `convert`/`magick` silently produces a blank image).

**Tech Stack:** React (TSX component), raw SVG, Playwright MCP browser tool (screenshot-based rasterization), ImageMagick `magick` CLI (PNG→ICO packing and resizing only — never given raw SVG input).

## Global Constraints

- No new npm dependencies (no `sharp`/`resvg`) — spec requires this to avoid adding an image-processing dependency for a one-time asset generation task.
- Mark geometry/colors must match the design exactly: outer ring `#ff6b9d`, middle ring `#6bd8ff`, inner ring `#9b8cff`, viewBox `0 0 88 88`, radii `41.5` / `29.9` / `19.1`.
- `apple-touch-icon.png` must be a **plain full-bleed square with no pre-rounded corners** (iOS applies its own mask) — all other icon rasters (`favicon.ico`, `logo192.png`, `logo512.png`) use the dark rounded-tile treatment from design 2d.
- `src/engine/` must stay framework-free — not touched by this plan (all changes are in `src/components/`, `src/routes/`, and `public/`).
- Vitest in this repo only picks up `src/**/*.test.ts` (no `.tsx`/jsdom component tests exist) — per CLAUDE.md. `BrandMark.tsx` therefore has no automated unit test; verification is `npm run check` (Biome/type-check) plus a visual check via the dev server, not a Vitest test file.
